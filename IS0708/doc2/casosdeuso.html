Caso de uso 01: Autentificar al usuario

Objetivo en contexto

Acceder al sistema.

Precondiciones

Debe ser posible establecer la conexión con la base de datos.

Postcondiciones si éxito

Se accede al sistema.

Postcondiciones si fallo

Se piden de nuevo los datos.

Actores

Usuario.

Secuencia normal

1- Introducir número de usuario.

2- Introducir contraseña.

3- Aceptar.

4- Acceder al sistema.

Secuencias alternativas

3a- Si el usuario no está en el sistema da un error y se da la opción a introducir de nuevo los datos.

3b- Si la contraseña no corresponde al usuario da un error y se da la opción a introducir de nuevo los datos.

Jefe de departamento.

Caso de uso 21: Modificar los datos de un empleado

Objetivo en contexto

Tener el sistema actualizado con los nuevos datos del empleado.

Precondiciones

El jefe de departamento ha accedido correctamente al sistema y ya está en su menú de opciones.

Postcondiciones si éxito

Los datos del empleado quedan actualizados.

Postcondiciones si fallo

Actores

Jefe de departamento.

Secuencia normal

1- Elegir opción de modificar.

2- Introducir los datos a modificar en lugar de los antiguos.

3- Confirmar.

4- Se actualizan los datos del empleado en la base de datos.

5- Se modifica el cuadrante si es necesario (CU-21) y se notifica al jefe de departamento.

Secuencias alternativas

2a- El empleado no está en el sistema. Comprobar que se ha escrito bien el identicador.

2b- Si los datos están en formato incorrecto la aplicación permite corregir los datos erróneos marcando los campos incorrectos de alguna forma (vuelta a 2).

2c- Se accede a algún dato que no se puede modificar. Modificar los datos permitidos o cancelar la operación.

Caso de uso 26: Crear/modificar cuadrante

Objetivo en contexto

Creación/modificación del cuadrante (considerando la creación como la modificación de un cuadrante vacío).

Precondiciones

Estar identificado, y debe ser posible establecer la conexión con la base de datos.

Postcondiciones si éxito

Cuadrante correcto.

Postcondiciones si fallo

Ninguna modificación.

Actores

Jefe de departamento y gerente.

Secuencia normal

1- Seleccionar opción modificar cuadrante.

2- Introducir datos si es necesario.

3- La aplicación propone un cuadrante alternativo.

4- Confirmación para creación de cuadrante.

5- Notificación del cuadrante a los usuarios afectados (CU-23).

Secuencias alternativas

4a.1a- El usuario rechaza la propuesta, y la apliación vuelve al punto 3.

4a.1b- La aplicación no puede generar otro cuadrante, pasa a CU-22.

4a.1b.1a- El usuario vuelve al principio para revisar las propuestas (vuelve al punto 3).

4a.1b.1b- El usuario cambia las condiciones iniciales para intentar generar otro cuadrante.
